I like BlueSky, but I _really_ like the idea of the AT Protocol. The idea of a federated web, a new protocol ground applications in. The promise of users being able to move their data between apps, and for apps to leverage a rich set of functionality, seems like this is fertile ground for the next generation of platforms.

Mastodon/ActivityPub got to the concept first, but from what I've read I'm not surprised it has problems with scaling.

Regardless I'm not used to thinking about things in a federated context. To find out the capabilities of this concept its best to just start building.

The AT Protocol team is pretty developer-friendly and have set up a nice web site to help ramp up new projects: https://atproto.com. There are examples up but they're all in React and NextJS. As an opening salvo to start learning the AT Protocol, I converted the [OAuth example](https://atproto.com/guides/oauth-tutorial) to use pure Svelte: https://github.com/cbroome/atpro-starter-svelte
